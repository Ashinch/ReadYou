### 備註

1. 每次冷啟動時都會自動檢查更新一次（要到下個版本發布後才能體驗到了）。

2. 檢查更新時，簡體中文語言使用 Gitee API，其他語言使用 GitHub API。

3. 請勿頻繁手動檢查更新（已配置了 2 秒的冷卻間隔），因為同一個 IP 每小時只能存取 60 次 GitHub API，而 Gitee API 的具體頻率限制尚不清楚（目前 8 小時了還沒重設）。

4. 閱讀頁面的重構在 0.8 版以後、圖示和封面的重構在 0.9 版以後、對 FreshRSS（Fever API、Google Reader API）的支援在 1.0 版以後。

5. 此版本因為上游解析套件的問題，導致只能單執行緒同步，可能會有些慢。該套件的作者表示下個版本會修正此問題，同時也將支援 Atom 協定。
